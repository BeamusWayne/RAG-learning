# Agentic RAG 学习笔记（本目录）

本目录用于系统学习 **Agentic RAG**（智能体驱动的检索增强生成）：让模型在**多步推理**中**自主决定何时检索、检索什么、如何修正答案**，而不是单次「问 → 检索 → 答」的流水线。

---

## 和传统 RAG 的差别（一句话）

| | 传统 RAG | Agentic RAG |
|---|-----------|-------------|
| 流程 | 固定：检索 → 拼进 prompt → 生成 | **动态**：可多次检索、改写查询、调用工具、自我检查 |
| 决策 | 由工程代码写死 | 由 **Agent / 图（Graph）** 在运行时决定 |
| 适用 | 简单问答、文档即答案 | 复杂问题、需多跳、需验证或纠错的场景 |

---

## 建议学习顺序

1. **先巩固基础 RAG**  
   分块、嵌入、向量库、Top-K 检索、重排序（若需要）。没有这块，Agent 只是在「空转」。

2. **再理解「Agent」在 RAG 里做什么**  
   - 何时检索（信息够不够？）  
   - 检索失败时是否换关键词 / 缩小范围  
   - 是否要把答案与引用对齐（grounding）  
   - 是否需要二次检索或拒绝回答  

3. **最后选一种编排方式落地**  
   - **状态机 / 图**：步骤清晰、易调试（例如 LangGraph 与本仓库 `LangGraph/` 教程）。  
   - **工具调用**：把检索、摘要、计算器封装成工具，由模型选工具。  

---

## 核心概念速查

- **检索（Retrieval）**：从知识库取相关片段；Agentic 场景里常被调用多次。  
- **查询改写（Query rewriting）**：把用户问题改成更适合检索的表述。  
- **规划（Planning）**：先拆子问题再分别检索，适合多跳问答。  
- **自我验证 / 反思（Reflexion）**：生成后对照检索结果检查是否矛盾或缺证据。  
- **工具（Tools）**：检索、搜索、计算、代码执行等；Agent 通过工具扩展能力。  
- **信任边界**：模型可能编造；Agentic RAG 更要强调**引用片段**与**引用不足时拒答**。
