# 05 · 向量库与索引（Vector Store）

把向量存起来 + 让"找最近邻"在毫秒级完成。
索引结构选错，要么慢成 PPT，要么精度悄悄掉一截你都不知道。

---

## 一句话理解

> 向量库 = "**存得下** + **查得快** + **查得准**"。
> 三者**不可兼得**，索引算法决定你的取舍点。

---

## 主流向量库对比

| 库 | 部署 | 索引 | 元数据过滤 | 适用 |
|---|---|---|---|---|
| **FAISS** | 库（无服务） | Flat / IVF / HNSW / PQ | 弱 | 学习、单机原型 |
| **Chroma** | 嵌入 / 本地 | HNSW | ✓ | LangChain 默认，原型友好 |
| **Qdrant** | 服务（Rust） | HNSW + scalar quant | ✓✓ | 生产中小规模 |
| **Milvus** | 服务（Go） | 多种 | ✓✓ | 生产大规模、多租户 |
| **Weaviate** | 服务 | HNSW | ✓✓ + GraphQL | 模块化、混合检索 |
| **pgvector** | Postgres 扩展 | IVFFlat / HNSW | ✓✓✓ SQL | 已有 Postgres 的项目 |

## 索引算法的三角权衡

```
              精度
               ▲
          Flat │
               │
         HNSW  │
               │
         IVF   │ (调 nprobe)
               │
         PQ    │
               └──────────► 速度 / 内存
```

| 算法 | 思路 | 精度 | 速度 | 内存 |
|---|---|---|---|---|
| **Flat** | 暴力线性扫描 | 100% | 慢 | 高 |
| **IVF** | 先聚类成 N 桶，查时只扫 nprobe 个桶 | 调 nprobe | 中-快 | 中 |
| **HNSW** | 多层小世界图，跳表式跳跃 | 高（默认选） | 快 | 高 |
| **PQ** | 向量乘积量化压缩 | 中 | 极快 | 极低 |
| **HNSW+PQ** | 图 + 压缩组合拳 | 中-高 | 极快 | 低 |

---

## 关键参数

### HNSW
- `M`：每层连接数（16–48）。大 → 召回高、内存高
- `efConstruction`：建索引探索深度（100–500）
- `efSearch`：查询探索深度（与召回精度强相关，运行时可调）

### IVF
- `nlist`：桶数（一般 `sqrt(N)` ~ `4*sqrt(N)`）
- `nprobe`：查询扫描桶数，**召回率最关键的旋钮**

---

## 元数据过滤

> 召回常常 = 向量近似 + 元数据精准过滤（时间、来源、租户、权限）。

**两种实现策略**：
1. **后过滤（post-filter）**：先 ANN 取 top-K，再筛选 → K 不够时召回大幅下滑
2. **前过滤 / 同步过滤**：在索引内直接限定子集（Qdrant / Milvus 支持）→ 推荐
