# 03 · 切分质量验证（Chunking Evaluation）

切分策略选好之后，**怎么知道它好不好**？这是 RAG 教程里最容易被跳过的一环——
大多数人切完直接进入"调 prompt 玄学循环"。本模块给你客观指标。

---

## 一句话理解

> 切分质量 = "**该在一起的没被拆开**" + "**该分开的没被混在一起**" + "**chunk 能独立被理解**"。
> 三件事都可量化。

---

## 三大评估维度

### 1. 边界泄漏（Boundary Leakage）

> 金标答案的关键句被切到两个 chunk 的边界处。

```
原文：... 公司 2023 营收 / 12.4 亿元，同比增长 ...
切分：[chunk A: ... 公司 2023 营收]
      [chunk B: 12.4 亿元，同比增长 ...]
召回 A → 答不出数字；召回 B → 不知道是谁。
```

**指标**：在合成 QA 集上，统计"金标答案跨 chunk"的比例。

### 2. 语义内聚度（Cohesion）

> 同一个 chunk 内部句子是否谈同一件事。

**指标**：chunk 内句对 embedding 余弦相似度的均值 / 方差。
方差越高 → chunk 越"杂"。

### 3. chunk 召回率（Chunk Recall）

> 用合成 QA 测：问题对应的"应命中 chunk"是否进入了 top-K。

**指标**：`Recall@K`（K = 5/10/20），按问题平均。

---

## 评估流程

```
原始语料
   │
   ├─→ 切分策略 A ─┐
   ├─→ 切分策略 B ─┼─→ 同一份合成 QA → Recall@K / 边界泄漏率 / 内聚度
   └─→ 切分策略 C ─┘
                       │
                       ▼
                选出帕累托最优策略
```

---

## 合成 QA 数据集怎么做

1. 从语料里随机抽 N 个段落
2. 用 LLM 为每段生成 1–3 个事实型问题 + 答案
3. 标注"应命中段落 id"
4. 人工抽检 ≥10%（关键步，否则垃圾进垃圾出）

> 没人工抽检的合成集会高估所有指标。
